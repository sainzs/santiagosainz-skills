# Visual Comparison

Use this when comparing two UI states, screenshots, mockups, or design references.

## Order

1. Layout and hierarchy: structure, grouping, content order, focal point.
2. Spacing and alignment: margins, padding, column widths, rhythm, component alignment.
3. Typography: size balance, line length, emphasis, text density.
4. Color and emphasis: contrast, accents, active/error/success/warning states.
5. Components and interaction cues: buttons, inputs, cards, tables, hover/focus/selected states.

## Output

- Overall read: one short paragraph.
- Biggest visible changes: ordered by user impact.
- Likely causes: clearly marked as hypotheses.
- Recommended fixes: concrete and prioritized.
- What to verify next: screenshots, states, breakpoints, or accessibility checks.

If only one image is available, switch to critique mode and say that explicitly.
